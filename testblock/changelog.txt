            Phase	        Cycles	    Time @ 100MHz	% of IP time
            Load x	        ~128	    ~1 μs	        <0.01%

            Output Q4K	    ~2.1M	    ~21 ms	        <0.44%
            Output Q6K	    ~5.5M	    ~55 ms	            —
    note that Q4k and Q6k are mutually exclusive: only one runs
    depending on the layer index. we can say that on average
    the output block is ~65ms
            
    other note: the gate phase first dequantizes X1 and X2 into float (multiply int8 by X12_QUANT_SCALE)
    then silu is applied to x1 with a lookup table (idx is computed from z) and float multiply 
    SiLU(z) = z × sigmoid(z) multiplication after the lookup. 
    then elementwise multiplication by X2
    and then requantize to int8
    it is much faster than X1/X2 because it is just two sequential passes over 8192 els with II=1
    so no weight memory to read, no outer loop, it is pure arithmetic on bram data
    and the bram is already there. 8192 INT8 values = 8 KB total
    There is no external memory access so Pure arithmetic: 8192 iterations x II=1 x 2 passes = ~16K cycles
    we can possibly take inspiration from this to speedup the parts that take long. 

5. start with output layer (but the problem is essentially the same for also X1 and X2)
    because they are coded in the same way. 
    the main problem is we need to read the weights from external ddr. 
    Q4K: 2048 rows × 4608 bytes/row = 9 MB streamed from DDR
    Q6K: 2048 rows × 6720 bytes/row = 13.1 MB streamed from DDR
    and the outer loop (for 2048 rows)
    theoretical maximum of ddr bandwidth: 100MHz x 16 bytes = 1.6GB/s
    for q4k 9MB/ 1.6GB/s = 5.6 ms just for weight loading. 
    also the MAC computation dominates because the weight loading and MAC op
    are sequential per row: load row -> mac row -> load nex row
    CLAUDE : inner vs outer loop of the output function
                OUTER: for (int out_i = 0; out_i < 2048; out_i++) {   ← one output element
                    load_row_down_q4k(W_down, out_i, row_buf);          ← fetch 4608 bytes from DDR
                    INNER: for (int n = 0; n < 256; n++) {              ← dot product over 8192 elements
                        #pragma HLS PIPELINE II=1
                        for (int bs = 0; bs < 8; bs++) {                ← 8 blocks unrolled
                            #pragma HLS UNROLL
                            ... MAC ...
                        }
                    }
                }
    the outer loop iterates over output rows (2048 output elements):
    for each row the all the weights are fetched, and then used in the inner loop mac op. 
    load and mac are seuqntial because the inner loop cannot start 
    before the row is fully loaded.
