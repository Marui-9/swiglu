Here’s what the old ggml-cpu_before_rewrite.c does for the SwiGLU offload:

Interface & memory layout (m_axi, no DMA for SwiGLU)

The IP’s control bus is at SWIGLU_IP_BASE = 0xA0020000.
Control regs: AP_CTRL/GIE/IER/ISR plus 64‑bit addr pairs for W, V, Wd, X, OUT, 
    and scalars MODE (Q4_K/Q6_K) and XSCALE.
Data is staged in udmabuf0 (single, contiguous CMA buffer). Fixed offsets:
SWG_VEC_OFF = 0x06C50000 (x, INT8 or F32 depending on quant step)
SWG_OUT_OFF = 0x06C60000 (output F32, 2048 floats)
Per‑layer cached weights (never overwritten):
W_gate base 0x06D00000, stride 0x00900000 (Q4_K size)
V_up base 0x0FD00000, stride 0x00900000
W_down base 0x18D00000, stride 0x00E00000 (fits Q6_K)
SWG_OUTPUT_SIZE = 8192 bytes (2048 floats).
Max batch is hard‑coded to 1 (SWG_MAX_BATCH = 1).
Init paths

init_hardware_offload() maps /dev/udmabuf0, records udmabuf_phys_base, 
                        opens sync_for_device/sync_for_cpu, and (legacy) 
                        sets up AXI-DMA descriptor chains, these DMA descriptors are not 
                        used by the SwiGLU path (the comment says “m_axi interface — no DMA”).
init_swiglu_offload() opens /dev/mem, mmaps the CTRL regs, enables GIE/IER, finds the UIO for 
                        SWIGLU_IP_BASE via sysfs, opens /dev/uioN, and arms interrupts (uio_enable write).
                        Runtime offload trigger

Hook is inside ggml_compute_forward_mul_mat: it intercepts only the down‑projection matmul when it sees 
src0 shape [8192,2048] with Q4_K/Q6_K and src1 F32, and only for batch==1. It walks the graph to recover 
W_gate, W_up, W_down, and the input x. If types are unexpected or batch>1, it falls back to CPU.

Weight cache
    Layer index = swiglu_call_count % 16. For each layer, W/V/Wd are copied once into their udmabuf slots; 
    subsequent tokens reuse the cached physical addresses.

Quantize x
Scans abs‑max of the 2048‑vector, computes x_scale, quantizes to INT8, writes to SWG_VEC_OFF, stores x_scale 
to the XSCALE register.

Register programming
Writes phys addresses for W, V, Wd, X, OUT; sets MODE (Q4_K/Q6_K derived from nb[3] size) and XSCALE; then ap_start=1.

Wait for completion
Prefers interrupt: poll() on swiglu_uio_fd; on wake, read() to ack, writes uio_enable to re‑arm, clears ISR.
If no UIO, it polls AP_CTRL with a timeout (logs timeout and aborts on 7s).

Output
Copies result from udmabuf_vptr + SWG_OUT_OFF into the ggml destination tensor (F32).

What’s not used for this path
The AXI-DMA descriptor setup present in init_hardware_offload() (RX/TX chains, payload splits) is legacy and unused by 
the m_axi SwiGLU IP. The SwiGLU IP itself masters the DDR via m_axi using the physical addresses programmed in the CTRL regs; 
there’s no SG DMA transfer for this kernel.

Threading
Offload runs in the single intercepted matmul; other threads sit at the ggml barrier. Interrupt mode avoids CPU spin while the FPGA runs.
This is the behavior you’re carrying over to the new fused kernel; you can drop the unused DMA descriptor code if the IP truly only needs 
m_axi + UIO interrupt.

DATAFLOW
Intercept trigger
It hooks inside ggml_compute_forward_mul_mat when it sees the down-projection matmul: W_down shape [8192, 2048], type Q4_K/Q6_K, and input (gated) type F32. Batch must be ne[1]==1.

Lazy init
On first hit (thread 0), it maps udmabuf0, records its physical base, and mmaps the SwiGLU CTRL regs at SWIGLU_IP_BASE. It also opens the UIO device for the IP and arms the ap_done interrupt.

Layer/mode selection
layer = swiglu_call_count % 16.
Quant mode (Q4_K vs Q6_K) is derived from src0->nb[3] (buffer size), not from the type enum.

Graph walk to collect operands
Finds gate matmul (W_gate), up matmul (W_up), and the input vector x upstream of the fused GLU node. Types must be Q4_K for W/V and F32 for x.

Staging / caching into udmabuf
One-time per layer: copies W_gate → SWG_LAYER_W_OFF(layer), W_up → SWG_LAYER_V_OFF(layer), W_down → SWG_LAYER_WD_OFF(layer). Flags remember if a layer’s weights are already loaded.

Quantize x
For the current token (batch=1), scans abs-max, computes x_scale, quantizes x to INT8, and writes to SWG_VEC_OFF.

Program accelerator (all via m_axi master, no DMA SG engine)
Computes physical addresses for W, V, Wd, x, out by adding offsets to udmabuf_phys_base.
Writes W/V/Wd/x/out addresses, MODE (Q4/Q6), and XSCALE into the CTRL register map.
Starts the IP (ap_start).

Wait for completion
Prefers interrupt: poll() on the UIO fd, read() to ack, then re-arms. If no UIO, it falls back to polling AP_CTRL with a timeout.

Output back to ggml
Copies the accelerator’s F32 output from SWG_OUT_OFF into the ggml destination tensor (dst). Logs a sample (out[0]) and increments swiglu_call_count.

What’s not involved
No AXI-DMA or SG descriptors are used for this path; the IP fetches/stores directly via its m_axi master ports using the physical addresses programmed above.
So, inputs (x, W_gate, W_up, W_down) are copied/quantized into fixed udmabuf slots, the IP reads them over m_axi, writes results back to SWG_OUT_OFF, and the code memcpy’s that buffer into the ggml tensor.

udmabuf itself isn’t a DMA engine; it’s a pinned, contiguous chunk of physical memory exposed to user space. You map it so you can memcpy data in/out, and you program its physical addresses into the FPGA. The accelerator then uses its own m_axi master ports to read/write that memory. So:

udmabuf = reserved, contiguous, physically-backed buffer (stable phys addresses).
