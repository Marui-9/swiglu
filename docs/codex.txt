Targeted bottleneck review
A) Primary latency bottlenecks (compute-side)
X1/X2 matvecs dominate front half

Each is ~2.71M cycles; even with overlap, one full X path remains on critical path.

Root cause: 8192 rows ├Ч (72 load + 256 MAC) structure.

Output stage dominates second half

Q4: ~1.64M cycles due largely to 512-cycle folded MAC loop per row.

Q6: ~2.96M cycles due decode loop + unroll factor 8 limit.

Root cause explicitly documented: routing relief via 2├Ч16 fold in Q4 path.

compute_gate is not huge, but hard lower bound is 2 passes

Two full FFN sweeps are required by current max-abs then quantize design.

~16k cycles is small vs matvecs, but architecturally fixed unless quantization method changes.

B) Potential II/timing stress points
Q6 decode FP accumulation revisit

You already use 8-lane partial accumulators (acc[n&7]) to satisfy revisit-period constraints for FP add latency; thatтАЩs good practice for II=1 retention.

Q4 down full partition fanout

32-block partitioning with wide unroll can create severe routing pressure; your fold to 16-wide groups is a deliberate congestion mitigation.

This is likely a key place where Fmax or route-congestion limits throughput scaling.

Large on-chip buffers in dataflow graph

X1_cache/X2_cache are large arrays with stream pragmas; real implementation may become ping-pong RAM rather than true small FIFOs depending on access pattern.

If scheduling inserts extra buffering, BRAM pressure and latency can increase.

D) Highest-impact optimization targets (ranked)
Cut per-row cycles in X1/X2

Biggest total latency lever.

Options: more parallel block processing, multi-row processing, or lowering per-row load overhead via caching/reuse (if multiple tokens).

Q4 down path: reduce 512-cycle folded MAC

If route allows, move from 2├Ч16 fold toward 1├Ч32 (or compromise 4├Ч8) to trade congestion vs latency.

Q6 down path: increase effective block parallelism

UNROLL factor=8 is likely the limiting factor; factor 16 could reduce row cycles if resources/timing permit.

Re-architect gate quantization for overlap

Current 2-pass full-vector quantization is clean/robust, but prevents some deeper streaming opportunities.
