Accuracy Degradation (Activation Quantization)Quantizing the FP32 activation vector $x$ to INT8 introduces quantization noise. However, dynamic symmetric quantization per token minimizes this error.Mechanism: The operation calculates $x_{scale} = \frac{\max(|x|)}{127}$ for each individual inference step prior to the DMA transfer.Impact: The resultant perplexity degradation is typically less than 0.5%. This is the standard mathematical trade-off required to bypass the LUT logic bottleneck and achieve the $>4$ tokens/second throughput target on the ZU5EV silicon.2. Model Weight IntegrityThe model weights are not changed. The GGUF file inherently stores weights in Q4_K and Q6_K block-quantized integer formats.Previous Datapath: The IP extracted the 4-bit integer weights and cast them to FP32 to execute an $FP32 \times FP32$ multiplication against the activation vector.Updated Datapath: The IP retains the 4-bit weights as integers, utilizes the INT8 quantized activation vector, and executes an $INT8 \times INT4$ multiplication natively within the DSP48E2 pre-adders. The FP16 block scales (d and dmin) are mathematically factored out and applied post-accumulation.The physical weights residing in the DDR memory remain strictly identical to the original model file.
